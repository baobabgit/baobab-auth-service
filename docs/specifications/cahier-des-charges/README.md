# Cahier des charges (entrée brute)

Dépose ici le **cahier des charges détaillé** du projet (Markdown, PDF, etc.).

C'est l'**entrée humaine figée**. Le rôle **Product Owner** la transforme en découpage
structuré et traçable sous [`../us/`](../us/) : User Stories (`US-XXX`) → Features
(`FEAT-XXX.Y`), chacune avec critères d'acceptation et identifiants.

- **Ici** : la source (le « d'où ça vient »).
- **Dans `../us/`** : la spec dérivée en reStructuredText (le « ce qu'on en a fait »).

> Ne modifie pas le contenu déposé ici pour « corriger » la spec : si une clarification
> est nécessaire, elle se note dans les `us/*.rst` dérivés, pas dans la source.
